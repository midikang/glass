# Glass Interactive Course

## What This Is

An interactive, single-page HTML course that teaches how the Glass codebase works—designed for "vibe coders" who build with AI but want to understand what's happening under the hood.

## How to Use It

1. Open `glass-course.html` in any modern web browser
2. Scroll through 6 modules that progressively explain Glass's architecture
3. Interact with:
   - Code ↔ English translation blocks (real code with plain-language explanations)
   - Animated group chat visualizations showing component communication
   - Interactive quizzes testing practical understanding
   - Glossary tooltips for every technical term
4. Use keyboard arrows to navigate between modules
5. Works completely offline—no dependencies except Google Fonts

## What You'll Learn

### Module 1: What is Glass?
- The product overview: what Glass does and why
- Real user journey: meeting recording → AI recall
- The two main features: Ask (context-aware AI) and Listen (meeting intelligence)

### Module 2: Meet the Main Actors
- The key components: Main Process, Window Manager, Listen Service, Ask Service, Database, Frontend
- Why this architecture (Electron + Next.js)
- How components communicate via event bridges

### Module 3: How Data Flows Through Glass
- Complete data flow: mic → transcription → database → AI
- Animated conversation showing components talking to each other
- Repository pattern: how Glass swaps between SQLite and Firebase

### Module 4: AI Providers & External Services
- All supported AI providers: OpenAI, Claude, Gemini, Ollama, Deepgram, Whisper
- Factory pattern for swapping providers
- How streaming responses work (see words appear as AI types)

### Module 5: Clever Engineering Patterns
- Repository Pattern (swap storage backends)
- Event Bridge (decouple components)
- Service Layer (business logic isolation)
- Graceful Shutdown (cleanup on exit)

### Module 6: The Complete Architecture
- Full system diagram: Desktop → Main Process → Frontend → Storage → External Services
- Complete request flow traced step-by-step
- Why this architecture makes Glass easy to debug, extend, and test

## Key Features

### Learning-Focused Design
- **Vibe coder friendly**: Assumes zero CS background, explains every concept
- **No jargon without definition**: Every technical term has a tooltip
- **Practical, not academic**: Teaches what helps you debug, extend, or steer AI better
- **Show, don't tell**: 50%+ visual content—diagrams, animations, code translations

### Interactive Elements
- ✅ Code ↔ English translation blocks (actual Glass code with line-by-line explanations)
- ✅ Animated group chat (components "talking" to each other like iMessage)
- ✅ Quizzes that test application, not memorization
- ✅ Glossary tooltips for every technical term
- ✅ Pattern cards highlighting engineering techniques you can steal
- ✅ Flow diagrams showing data journeys

### Technical Implementation
- **Single self-contained HTML file** (86KB)
- **No dependencies** except Google Fonts CDN
- **Fully responsive** (desktop, tablet, mobile)
- **Keyboard navigation** (arrow keys to move between modules)
- **Warm, distinctive design** (off-white backgrounds, vermillion accent, Bricolage Grotesque display font)
- **Scroll-based navigation** with progress tracking
- **Works offline** after first load

## Design Philosophy

This course inverts traditional CS education:

**Old way**: Memorize concepts for years → eventually build something → finally see the point (most quit before step 3)

**This way**: Build something → experience it working → now understand how it works

The learner already has context—they've used Glass or want to understand it. The course meets them where they are: "You know that Listen button? Here's what happens under the hood when you click it."

## Customization

To adapt this course for another project:

1. **Change the accent color** (`--color-accent` in CSS)
2. **Replace modules** with your project's architecture
3. **Update code examples** with actual code from your codebase
4. **Modify actor colors** to represent your components
5. **Keep the patterns**: translation blocks, quizzes, tooltips, animations
